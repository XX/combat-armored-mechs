# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Характер проекта

Это **не программный проект**, а художественный воркбилдинг-проект (мир для научно-фантастических историй). Содержимое — на русском языке. Здесь нет сборки, линтеров и тестов; «исходник» — это Markdown-документы с описанием мира и сюжетов.

Единственный содержательный файл — `Боевые роботы.md`: библия мира (вселенная боёв с применением роботов в недалёком будущем). Все правки делаются в Markdown и фиксируются через git.

## Канон мира (с чем нужно быть консистентным)

При написании или редактировании любого текста соблюдай установленные в `Боевые роботы.md` принципы — они задают тон и ограничивают допущения:

- **Строгая НФ / технотриллер.** Только технологии, известные сейчас или правдоподобные в ближайшем будущем. Допускаются небольшие фант-допущения, но **никакой «магии»** и никакого сильного ИИ как у классических фантастов.
- **Отличие от Battletech/MechWarrior** (источник вдохновения): вместо империй — корпорации и «цифровые анклавы»; ключевая ставка — производство ресурсов прямо на поле боя и биомеханика/уязвимость пилота.
- **Социально-политическая модель — часть мира, не фон.** Единое мега-общество «Человечество» без формальных границ; государства выродились в культурно-административные сообщества; ресурс — не территория, а производственные/вычислительные мощности, энергетика, логистика, данные и влияние.
- **Войны «со связанными руками»**: ведутся ради контроля инфраструктуры, а не на уничтожение; исход = арбитраж («кто выиграл битву, тот и прав»). Сохранение производства и логистики важнее числа уничтоженных машин.
- **Экономика на энергии**: валюта — Джоуль (Дж), не доллар.
- **Информационное превосходство** (РЭБ, кибератаки, рои микродронов) часто решает бой раньше визуального контакта.
- Пилотируемые роботы дают пилоту **обратную связь болью** — намеренная уязвимость, а не баг.

Когда добавляешь технологию, оружие, класс ББР или сюжет — проверяй, что он не противоречит этим принципам и существующим разделам. При конфликте указывай на него, а не «тихо» переписывай канон.

## Структура документа `Боевые роботы.md`

Документ организован по разделам `##`; держись этой структуры при добавлении материала:

`Черты мира` → `Технологии` (Энергетика / ИИ / Производство / Материалы / Логистика) → `Боевые роботы` (включая классы ББР: лёгкие, средние, тяжёлые, сверхтяжёлые; специализированные и ройные системы) → `Вооружение` (Кинетическое / Ракетное / Энергетическое) → `Особенности войны` → `Космические боевые действия` → `Возможные сюжетные темы`.

`Возможные сюжетные темы` — это банк идей-зацепок (по одному буллету на идею); новые идеи добавляй туда же в том же формате.

## Соглашения

- Язык всего контента — **русский**; сохраняй стиль и регистр существующего текста.
- Открытые/нерешённые вопросы помечаются знаком `(?)` в конце пункта — это намеренные заметки автора, не ошибки; не удаляй их без явной просьбы.
- В тексте встречаются опечатки (напр. заголовок «Боевыве роботы», «вооруение», «передвежные»). Исправляй опечатки только когда об этом просят или мимоходом в редактируемом фрагменте; не затевай массовую вычитку без запроса.
- Списки — маркированные (`-` или `*`), классы и сущности оформляются короткими подпунктами с характеристиками. Следуй этому при добавлении новых сущностей.
